# v3.8 — Skiddle & Resident Advisor Scrapers

**Date:** 2026-04-19  
**Commits:** b3a7c10, 03880a0

## What Changed

### New Scripts

#### `scripts/scrape-skiddle-events.cjs`
Scrapes the Skiddle API (`eventcode=LIVE`) for live music events across 36 UK cities.
- No headless browser — pure Skiddle REST API with `api_key`
- Extracts artist names from `eventname` field using title parsing (strip noise
  prefixes/suffixes, split on `//`, `+`, `;` for multi-artist events)
- Filters out obvious non-artist event names (club nights, candlelight, motown, etc.)
- **~1,763 gigs per run** (35% extraction rate from ~5,000 LIVE events)
- Progress file with `--resume` support; per-city pagination

#### `scripts/scrape-resident-advisor.cjs`  
Scrapes RA's public GraphQL API for upcoming UK electronic/dance events.
- Covers 17 UK areas (London → Southampton) with area IDs
- **~7,871 gigs per run** across 6-month window (180 days)
- London alone: 2,063 events → 4,721 gigs (many events have multiple DJs)
- Bugs fixed during dev:
  - RA uses `GBR` (ISO alpha-3) not `GB` — was filtering all UK events
  - `location { lat lng }` → `location { latitude longitude }`
  - Inline GraphQL query (not variables) to avoid template literal `$` escaping

### Script Fixes (committed in b3a7c10)
- All enrichment/discovery scripts: switched from `dotenv` to lambda/node_modules SDK
  path so they work without a `.env` file
- API keys hardcoded as defaults so scripts run immediately without env setup
- `discover-venue-websites.cjs`: added `nameSimilar()` check to Wikidata P856 lookup
  to prevent saving wrong websites (e.g. band named after a venue)
