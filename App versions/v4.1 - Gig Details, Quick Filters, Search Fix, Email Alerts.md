# v4.1 — Gig Details, Quick Filters, Search Fix, Email Alerts

**Date:** 2026-04-22
**Commit:** f510b9f

## What shipped

### Gig detail pages (`/gigs/[id]`)
- Full ISR page per gig with `getStaticPaths fallback: blocking`
- Ticket price comparison table (cheapest seller highlighted in green)
- Follow artist button inline
- Breadcrumb: Gigs → Artist → Date
- Venue card links back to venue page
- Artist sidebar card links to artist page
- Share section: copy link + Twitter share
- Past event badge, sold out state, free entry handling
- `GET /gigs/:id` endpoint added to API Lambda

### Quick date filters on /gigs
- Three new preset buttons above price filters: **Today**, **This week**, **Weekend**
- Sets `from`/`to` range — highlighted in violet when active
- Works alongside all existing city/genre/price filters

### Search rewrite
- Previously loaded all 35,000 artists + 10,000 venues client-side on every page load
- Now calls `/search?q=` API endpoint with 300ms debounce
- Instant spinner feedback, no empty state flicker
- Results appear within one round-trip instead of after a 3-4s bulk load

### Email alerts — Notify Lambda
- Switched from SES to Resend (uses Node.js built-in `https`, dropped `@aws-sdk/client-ses` dep)
- `RESEND_API_KEY` env var now set on notify Lambda
- DynamoDB stream trigger already connected to `gigradar-gigs` table
- Emails now include link to gig detail page
- Lambda deployed to `gigradar-notify`

### GigCard
- Date block now links to `/gigs/[id]` gig detail page

## Already done (ISR on artist/venue pages)
- `src/pages/artists/[id].jsx` and `src/pages/venues/[slug].jsx` both already had
