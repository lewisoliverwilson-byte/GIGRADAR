# GigRadar ‚Äî Version 1.3: Live Gig Scraper Engine

**Date:** 2026-04-03
**Deployment:** Pushed to `main` ‚Äî Amplify redeploy (commit `6e6ce4c`). AWS infrastructure provisioned live via CLI.

---

## Overview

Replaced all static mock gig data with a live scraping pipeline. Every 4 hours, a Lambda function fetches the top 1000 UK artists from Last.fm and all upcoming UK music events from Ticketmaster, matches them, and stores the results in DynamoDB. The frontend fetches from an API Gateway endpoint on load and replaces the static arrays from `data.js`, falling back to mock data until the first scrape has run.

## What Changed

### New: `lambda/scraper/index.js`
- Fetches top 1000 UK artists from Last.fm `geo.getTopArtists` (20 pages √ó 50, 250ms between requests)
- Fetches all upcoming UK music events from Ticketmaster (paginated at 200/page, 220ms between requests ‚Äî stays under 5 req/sec limit)
- Normalises artist names (strips "the", removes punctuation) for cross-source matching
- Upserts artists to `gigradar-artists`; upserts matched gigs to `gigradar-gigs`
- Updates `upcoming` count on each artist after each run

### New: `lambda/api/index.js`
- Routes: `GET /artists`, `GET /artists/:id`, `GET /artists/:id/gigs`, `GET /gigs`
- All responses include CORS headers (`Access-Control-Allow-Origin: *`)
- `/gigs` returns all upcoming gigs sorted by date (scan with date filter, limit 200)

### `public/js/config.js`
- Added `apiBaseUrl: 'https://8joy7er006.execute-api.us-east-1.amazonaws.com'`

### `public/js/app.js`
- Added `loadLiveData()` ‚Äî called first in `init()`, fetches `/artists` and `/gigs` in parallel, replaces `window.ARTISTS` and `window.GIGS` if the API returns data
- Silent fallback to `data.js` mock data if API is unreachable or returns empty

## Activation Required

The scraper will not run until two API keys are added as Lambda environment variables:

```
LASTFM_API_KEY        ‚Äî register free at last.fm/api/account/create
TICKETMASTER_API_KEY  ‚Äî register free at developer.ticketmaster.com
```

Once set, the scraper can be triggered manually for the first run:
```bash
aws lambda invoke --function-name gigradar-scraper --region us-east-1 response.json
```

## Known Issues / Limitations

Carries forward all issues from v1.2, plus:

1. **API keys not yet set** ‚Äî scraper will log "No API key" and return empty results until environment variables are added.

2. **Artist matching is exact-normalised only** ‚Äî artists whose names differ between Last.fm and Ticketmaster (e.g. abbreviations, featuring credits) will not be matched. Fuzzy matching can be added later.

3. **Ticketmaster only** ‚Äî no smaller/independent UK venue coverage yet. Skiddle API integration planned.

4. **No past gigs from live data** ‚Äî the scraper only fetches upcoming events. Historical gigs still use mock data from `data.js`.

5. **DynamoDB scan on `/gigs`** ‚Äî acceptable at current scale but should move to a GSI-based query as data grows.

6. **No deduplication across sources** ‚Äî when additional ticket sources are added, the same gig could appear multiple times.
