# v4.0 — Feature Roadmap

Planned features for the next major version.

## Connect Your Venue — Facebook OAuth Flow

**Problem:**
Facebook Events is the largest source of grassroots UK live music data (~40–60% of free/unticketed pub gigs exist only on Facebook). The Events Discovery API was shut down in 2018, and HTML scraping is blocked by Facebook's authentication wall.

**Solution:**
A "Connect Your Venue" feature where venue owners authorize GigRadar to read their Facebook Page's events via OAuth.

**How it works:**
1. Venue owner visits a GigRadar page and clicks "Connect Facebook Page"
2. Standard Facebook Login OAuth flow — they grant `pages_read_engagement` and `pages_events` permissions
3. GigRadar stores the access token and periodically reads events from that Page
4. Events are ingested into the gigs DB with `source: 'facebook'` and the Facebook event URL as the ticket link
5. For free/no-ticket events, the Facebook event URL is shown instead of a ticket seller

**Why this is the right approach:**
- Fully legitimate — no ToS violations
- Data is accurate and up to date (straight from the Page)
- Scales as venues sign up
- This is how Songkick and Bandsintown originally built grassroots coverage

**Scope:**
- Facebook App with `pages_read_engagement`, `pages_events` permissions (requires App Review)
- OAuth flow on venue profile/registration page
- Background job to sync events from connected Pages nightly
- Venue dashboard showing sync status and event preview
- For venues without Facebook: manual event submission form as fallback

**Notes:**
- Facebook App Review for these permissions can take 2–4 weeks
- Access tokens expire — need refresh token handling
- Events on Facebook may lack structured artist/performer data; may need name extraction from event title

## Connect Your Venue — Instagram OAuth Flow

**Problem:**
Many venues and artists announce free/unticketed gigs exclusively via Instagram posts and Stories. Instagram has no Events feature, so data is unstructured, but it's a primary promotional channel for grassroots music — especially smaller venues and unsigned acts.

**Solution:**
Extend the "Connect Your Venue" OAuth flow to also connect the venue's Instagram Business/Creator account, allowing GigRadar to read their posts and extract upcoming event info.

**How it works:**
1. Venue owner connects their Instagram account via the same OAuth flow as Facebook (Meta uses the same platform — one login covers both)
2. GigRadar requests `instagram_basic` and `pages_read_engagement` permissions
3. Background job reads the venue's recent posts and applies pattern matching / NLP to extract: artist name, date, and any price info ("free entry", "£5 on door")
4. Matched events are ingested with `source: 'instagram'` and the Instagram post URL as the link
5. Low-confidence extractions are flagged for manual review rather than auto-published

**Why this is worth doing:**
- Same App Review process as Facebook — no extra overhead once the Meta app exists
- Covers a huge volume of grassroots gig announcements that exist nowhere else
- Instagram is the primary channel for emerging/unsigned artists promoting shows

**Scope:**
- Uses the same Meta Developer App as the Facebook integration
- Additional permissions: `instagram_basic`, `instagram_content_publish` (for future venue notifications)
- NLP/regex layer to extract event data from post captions (artist name, date, venue, price)
- Confidence scoring — only auto-ingest high-confidence extractions
- Manual review queue in venue dashboard for ambiguous posts

**Notes:**
- Requires Instagram Business or Creator account (not personal accounts)
- Instagram Graph API only accesses posts, not Stories (Stories are ephemeral and unavailable via API)
- Meta App Review required — same process as Facebook, can be submitted together
- Post captions are unstructured; extraction will be imperfect — treat as supplementary data, not primary source
- Long-lived tokens available for Instagram (60-day expiry, auto-refreshable)
