# GigRadar — Version 2.2: Past Gigs & Full Artist Coverage

**Date:** 2026-04-04
**Deployment:** Pushed to `main` — both Lambdas updated directly via CLI (commits `2d698f5`, `33f8a80`, `86e3568`).

---

## Overview

Artist pages now show the past 12 months of gigs alongside upcoming ones. Bandsintown expanded from top 300 to all 1,000 artists. Setlist.fm added as a dedicated historical gig source with song-by-song setlist data. Scraper coverage is now substantially broader across all 10+ sources.

---

## What Changed

### `lambda/api/index.js`
- `getArtistGigs`: changed date filter from `>= today` to `>= yearAgo` (12 months back) — past gigs are now returned to the frontend

### `lambda/scraper/index.js`

**Bandsintown**
- Expanded from top 300 to all 1,000 artists
- Date range changed from `today → +12 months` to `-12 months → +12 months` in a single request per artist
- Now provides past and upcoming coverage for every artist on the list

**Songkick**
- Removed `date < today` filter — past events from gigography pages are now stored
- Date window: past 12 months to next 12 months

**Setlist.fm (new source)**
- New `fetchSetlistFm(artists)` function using the Setlist.fm REST API
- Queries all artists with a MusicBrainz ID (946 of 999)
- Stores setlist preview (first 8 songs) alongside venue and date
- Rate limit managed via once-per-day scheduling: timestamp stored in DynamoDB (`_gigradar_meta` record), subsequent runs within 20 hours are skipped automatically
- Uses full 1,440 req/day allowance in one daily pass — covers all 946 eligible artists
- Requires `SETLISTFM_API_KEY` env var (free at setlist.fm/api — key added)

### `src/pages/ArtistDetail.jsx`
- Already splits gigs into upcoming/past tabs — past gigs now populate automatically

## Known Limitations

1. Setlist.fm only covers artists with a MusicBrainz ID — 53 artists without one get no Setlist.fm data (Bandsintown/Songkick still cover them)
2. Setlist.fm rate limit is 1,440 req/day — if upgraded, remove the once-per-day gate and increase artist coverage further
3. Past gig ticket links from Setlist.fm show `available: false` (the gig already happened) — these are informational only
