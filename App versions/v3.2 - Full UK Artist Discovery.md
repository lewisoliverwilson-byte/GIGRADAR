# GigRadar — Version 3.2: Full UK Artist Discovery

**Date:** 2026-04-04
**Deployment:** Pushed to `main` (commit `28cb236`). Scraper Lambda updated directly via CLI.

---

## Overview

GigRadar v3.2 flips the scraper from artist-centric to venue-centric discovery. Previously, every source was filtered to the Last.fm UK top 1,000 chart — any act not in that list was invisible. Now, every bulk ticketing source auto-creates artist records for any UK act it encounters. The result: every band with a ticketing presence on Skiddle, Dice, Ticketmaster, RA, See Tickets, Gigantic, WeGotTickets, or Eventbrite will appear on GigRadar within one or two scraper runs.

---

## What Changed

### `lambda/scraper/index.js`

**New helpers:**
- `isGenericName(name)` — filters out non-act strings like "Live Music", "Open Mic", "TBA", "DJ Night" etc. that would pollute the artist DB
- `loadAllArtistsFromDb()` — scans the full `gigradar-artists` table at startup; adds all previously discovered grassroots acts to `nameMap` so they aren't re-seeded on every run
- `autoSeedArtist(name, nameMap)` — for any act not already in `nameMap`: creates a minimal DynamoDB record (`isGrassroots: true`, no rank, empty genres) and adds it to the in-memory map immediately; uses `if_not_exists` so existing enrichment data is never overwritten

**New function: `fetchTicketmasterBulk(nameMap)`**
- Bulk-scans all UK music events on Ticketmaster (up to 50 pages × 200 events = 10,000 events)
- Separate from the per-artist TM queries (which still run for Last.fm top 1,000)
- Calls `autoSeedArtist` for any act not yet in DB
- Stops early if TM reports no more pages

**Sources now in discovery mode** (nameMap guard removed):
- Skiddle — up to 3,000 UK events/run
- Dice — up to 2,000 UK events/run
- Resident Advisor — up to 1,000 UK events/run
- See Tickets — up to 1,500 UK events/run
- Gigantic — up to 1,000 UK events/run
- WeGotTickets — up to 1,000 UK events/run
- Eventbrite — up to 1,000 UK events/run

**New function: `enrichGrassrootsArtists()`**
- Runs after all gig scraping
- Scans for grassroots artists missing `monthlyListeners` (not yet enriched)
- Capped at 100 artists per run to stay within Lambda timeout
- For each: calls Last.fm `artist.getInfo` (listeners, MBID, bio), then Deezer (photo)
- Enriches 100 new acts per run — so a batch of 1,000 new acts is fully enriched within 10 scraper cycles

**Main handler:**
- Builds `nameMap` from Last.fm chart artists + all DB artists combined (previously: Last.fm only)
- Calls `fetchTicketmasterBulk` in the gig pipeline
- Calls `enrichGrassrootsArtists` at the end
- Logs total artist count split (charting vs grassroots)

## What This Means in Practice

| Run | What happens |
|-----|-------------|
| First run after deploy | Bulk sources discover new acts, create minimal DB records; their gigs are saved immediately; up to 100 enriched with Last.fm + Deezer |
| Subsequent runs | More grassroots acts enriched (100/run); previously discovered acts get ongoing gig updates |
| After ~10 runs | All discovered grassroots acts should have enrichment data |

## What's Still Out of Scope

| Gap | Reason |
|-----|--------|
| Truly self-promoted acts with no ticketing platform | Requires venue website crawler (Puppeteer/cheerio scraping venue "what's on" pages) — deferred |
| Facebook Events | Meta ToS; requires login; deferred |
| Acts only on Bandcamp/SoundCloud | No gig data to extract; deferred |
