# GigRadar — Version 2.3: Setlist Pagination & Ticket Deduplication

**Date:** 2026-04-04
**Deployment:** Pushed to `main` — Lambda updated directly via CLI (commit `0031758`).

---

## Overview

Two bug fixes: Setlist.fm now fetches 2 pages for the top 480 artists, recovering missing UK gigs (e.g. Oasis 2025 UK tour) that were pushed off page 1 by international dates. GigCard no longer shows duplicate ticket buttons when the same seller appears multiple times in the tickets array.

---

## What Changed

### `lambda/scraper/index.js`

**Setlist.fm — 2-page fetch for top 480 artists**
- Artists ranked 1–480 (by `lastfmRank`) now fetch pages 1 and 2 from Setlist.fm; artists ranked 481+ fetch page 1 only
- Total requests per daily run: 480×2 + 466×1 = **1,426** — stays within the 1,440/day free limit
- Root cause: Oasis (rank 60) had only 2 gigs stored because their entire July–August 2025 UK tour (Manchester, Cardiff, Edinburgh, Wembley) appeared on page 2 of Setlist.fm — page 1 was filled with international tour dates
- The `twoPageIds` Set is built by sorting `withMbid` by `lastfmRank` and taking the first 480

### `src/components/GigCard.jsx`

**Ticket button deduplication by seller**
- Replaced the plain `.filter()` on `tickets` with a `Map` keyed by seller name
- When the same seller appears twice, keeps the entry with a real price over `'See site'`
- Result: maximum one button per ticket site per gig

## AWS Infrastructure

No new resources or env vars.

## Known Limitations

1. Setlist.fm still only covers artists with a MusicBrainz ID (946 of 999)
2. Artists ranked 481–999 still get only 1 page — sufficient for most, but very active international artists at lower ranks may still miss some UK dates
3. The once-per-day gate resets on the next scheduled run (~4–8h from deployment); Oasis gigs will appear after that run completes
