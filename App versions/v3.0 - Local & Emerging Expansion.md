# GigRadar — Version 3.0: Local & Emerging Expansion

**Date:** 2026-04-04
**Deployment:** Pushed to `main` (commit `5742af1`). Both Lambdas updated via CLI.

---

## Overview

GigRadar v3 expands from a mainstream gig discovery tool into a destination for all UK live music — from arena headline acts to unsigned bands in pub back rooms. Key additions: a venue database auto-built from gig data, venue pages with follow support, a Discover page with city search and genre chips, a unified Search page covering both artists and venues, grassroots badges for emerging acts, and ticketing type badges (Free Entry, Pay on Door, Check Venue).

---

## What Was Built

### Infrastructure
- **New DynamoDB table: `gigradar-venues`** — PAY_PER_REQUEST, `venueId` as partition key

### `lambda/scraper/index.js`
- Added `toVenueId(name, city)` and `toVenueSlug(name, city)` helper functions
- Added `upsertVenues(gigsArr, today)` — extracts all unique venues from merged gig data and upserts them to `gigradar-venues` with name, city, slug, upcoming count, and active flag
- `canonicalVenueId` is now stamped on every gig record before it's written to DynamoDB (enables efficient venue→gigs lookup)
- Venue table is seeded automatically on every scraper run — no manual seeding needed

### `lambda/api/index.js`
- `GET /venues` — returns all active venues sorted by upcoming gig count
- `GET /venues/:slug` — returns a single venue by slug
- `GET /venues/:slug/gigs` — returns all gigs at a venue (past 12 months + upcoming), matched by `canonicalVenueId` or `venueName`

### `src/utils/api.js`
- Added `getVenues()`, `getVenue(slug)`, `getVenueGigs(slug)`

### `src/context/FollowContext.jsx`
- Added `followingVenues` Set alongside existing `following` Set
- Added `followVenue`, `unfollowVenue`, `isFollowingVenue` functions
- Venue follows persisted in localStorage under a separate key (`gigradar_following_venues_*`)

### New Components & Pages

**`src/components/VenueCard.jsx`**
- Square card with venue photo or coloured initial block
- Venue type badge (pub, club, theatre, etc.)
- Name, city, upcoming count
- Follow button (venue-specific, prompts sign-in if logged out)

**`src/pages/VenuePage.jsx`** — `/venues/:slug`
- Hero image or coloured block with venue name, city, capacity, type
- Follow venue button + website/Instagram links
- Upcoming / Past tabs, each showing a paginated GigCard list

**`src/pages/Discover.jsx`** — `/discover`
- City/town text search (free-form, filters gigs and venues)
- Genre chips (15 genres from PRD taxonomy) — filters gigs by artist genre
- Toggle between Gigs view and Venues grid
- **Grassroots badge** on gig cards for artists ranked >500 or not in the top charts
- Paginated gig list (20 per page)

**`src/pages/Search.jsx`** — `/search`
- Live search across all artists and all venues as user types
- Updates `?q=` URL parameter (shareable/bookmarkable)
- Artists section (grid of ArtistCards) + Venues section (grid of VenueCards)
- Empty state with message: "If they're a UK act, they may appear soon as we scan more venues"

### Updated Components

**`src/components/GigCard.jsx`**
- New ticketing status display logic:
  - `isFreeEntry: true` → green **Free entry** badge
  - `ticketType: 'door'` → amber **Pay on door** badge
  - Available ticket links → existing seller pill buttons (unchanged)
  - No tickets, not past, not sold out → **Check venue for tickets** text
  
**`src/components/Navbar.jsx`**
- Added **Discover** link
- Search button now navigates to `/search` instead of `/artists`
- Search placeholder updated to "Search artists, venues..."

**`src/App.jsx`**
- Added routes: `/discover`, `/search`, `/venues/:slug`

## What's Not in This Version (per PRD)

| Feature | Reason |
|---------|--------|
| Venue website crawler (Puppeteer/Playwright) | Requires headless browser infrastructure on Lambda/ECS — significant engineering work |
| Instagram/Facebook scraping | Legal concerns flagged in PRD; Meta ToS review required |
| Automated artist profile creation from venue scans | Depends on crawler not yet built |
| Artist profile claiming | Needs email verification flow; deferred to v3.1 |
| Genre tagging dashboard | Internal tool; deferred to v3.1 |
| Manual artist/venue additions | Deferred — current auto-extraction from gig data covers the existing dataset |

## Data Seeding Note

Venues are auto-extracted from gig data on every scraper run. The first run after this deployment (triggered 2026-04-04 ~19:23) will populate the `gigradar-venues` table from all existing gig records. Venue pages and the Discover venue grid will be populated after that run completes (~10–15 minutes). Subsequent runs keep the table up to date automatically.
