# v4.0 — Venue Follow, City Pages & Spotlight

**Date:** 2026-04-22
**Commit:** 685c179
**Branch:** main

## What shipped

### Expansion 1 — Follower count on venue pages
- `followerCount` field on venue records, atomically incremented/decremented via DynamoDB `ADD 1` / `ADD -1`
- Conditional guard prevents count going below zero on concurrent unfollows
- Displayed on venue page header alongside "Discovered X artists" badge

### Expansion 2 — Weekly digest email
- `scripts/send-weekly-digest.cjs` — sends upcoming shows (next 14 days) from followed venues
- Per-venue skip if no shows; "quiet week" fallback if all venues are quiet
- Wired into `weekly-refresh.sh` after data refresh (so show data is fresh before sending)

### Expansion 3 — "Discovered them first" stat
- Computed in `getVenueGigs` Lambda: past artists at the venue with >50k monthly Spotify listeners
- Uses `BatchGetCommand` on up to 100 past artists per venue
- Displayed as "⚡ Discovered N artists" amber badge on venue page

### Expansion 4 Phase 1 — Early Radar data collection
- `enrich-artists-spotify.cjs` now appends `{ ts, l }` entries to `listenersHistory` array on every enrichment run
- Phase 2 (homepage UI) ships once ≥2 data points exist per artist (minimum 1 week)

### Expansion 5 — City browse pages
- `/cities` index listing all 10 UK cities
- `/cities/[city]` — grassroots venues sorted Spotlight-first, then by followerCount
- Spotlight CTA banner links to `/list-your-venue`
- Cities: London, Manchester, Glasgow, Bristol, Birmingham, Leeds, Edinburgh, Liverpool, Brighton, Newcastle

### Expansion 6 — Merged follow UX
- `AlertButton.jsx` is now the single follow button for venues
- One-click follow if email stored in `gr_alert_email` localStorage (no form shown)
- Writes to `gr_followed_venues` localStorage in parallel for homepage personalisation
- Removed the separate localStorage-only Follow button from VenuePage

### Expansion 7 — Venue Spotlight
- `/list-your-venue` landing page — £49/mo, Stripe Payment Link, benefit bullets
- `scripts/set-spotlight.cjs <venueId> [--remove]` — admin script to activate/deactivate
- `scripts/send-spotlight-stats.cjs` — weekly email to Spotlight venues (follower count + upcoming gigs)
- Spotlight badge (★) shown on city pages, pinned to top of sort order

## Infrastructure
- RESEND_API_KEY set in `quick-refresh.sh` and `weekly-refresh.sh`
- Lambda API deployed with: followerCount increment/decrement, BatchGetCommand import, discoveredCount in getVenueGigs
- Stripe Payment Link URL is a placeholder — replace `STRIPE_PAYMENT_LINK` in `list-your-venue.jsx` with real link
