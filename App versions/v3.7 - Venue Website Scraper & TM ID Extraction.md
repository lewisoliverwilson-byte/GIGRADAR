# v3.7 — Venue Website Scraper & TM ID Extraction

**Date:** 2026-04-19  
**Commits:** da2e778, 2f7934c

## What Changed

### New Scripts
- **`scripts/scrape-venue-websites-full.cjs`** — Full venue website scraper (local, not Lambda).
  Crawls 813 real venue websites with multi-strategy event extraction:
  - JSON-LD MusicEvent / Event schema
  - Squarespace iCal feeds (fetches tiny 350-byte per-event iCal instead of 90KB pages)
  - Dice.fm direct event links (artist name parsed from URL slug)
  - Eventbrite individual event links (fetches JSON-LD from event pages)
  - HTML pattern matching fallback
  - Detects embedded platform widgets and saves to `ticketingUrls` for Lambda
  - `--venue="Name"` for single venue testing, `--dry-run`, `--debug`, `--limit`, `--workers`

- **`scripts/discover-venue-websites.cjs`** — Finds official websites for venues without one.
  Sources: Wikidata P856 (official website property), Wikipedia external links, MusicBrainz URL relations.

- **`scripts/extract-tm-ids-from-websites.cjs`** — One-time script: extracted 803 Ticketmaster venue IDs
  from incorrect `website` field entries (TM venue URLs were stored as websites). Saved to `tmVenueId`,
  cleared `website` field. Result: 803 more venues now covered by TM API scraping.

### Lambda Improvements (`lambda/scraper/index.js`)
- `VENUE_BATCH_SIZE`: 50 → 100 (full 5762-venue cycle: ~16 days → ~8 days)
- `fetchVenueWebsiteGigs`: upgraded extraction:
  - Handles `|`-separated titles: "A Lovely Time | Artist: Tour" → "Artist"
  - Squarespace iCal support (same approach as local scraper)
  - Dice.fm URL slug parsing for embedded Dice links
  - Skips ticketing platform URLs stored in `website` field

### Artist Title Extraction Fix
Updated `extractArtistFromTitle` to handle pipe-separated formats:
- Before: "A Lovely Time | Susie McCabe: Best Behaviour" → "A Lovely Time" ✗
- After: "A Lovely Time | Susie McCabe: Best Behaviour" → "Susie McCabe" ✓

## Data Changes
