# GigRadar — Version 2.1: Artist Photos

**Date:** 2026-04-04
**Deployment:** Pushed to `main` — Lambda updated directly via CLI (commit `e1ea5e9`).

---

## Overview

Added high-quality artist photos to all 1,000 artist cards and detail pages using the Deezer API (no API key required). Photos are fetched once and stored in DynamoDB — subsequent scraper runs skip artists that already have images.

---

## What Changed

### `lambda/scraper/index.js`
- Added `fetchDeezerImage(artistName)` — queries `api.deezer.com/search/artist` for a 500×500px artist photo
- Added `enrichArtistImages(artists)` — scans DynamoDB first to find artists without `imageUrl`, then calls Deezer only for those (saves ~150s on every subsequent run)
- Fixed `upsertArtist` — switched from `PutCommand` (overwrote all fields) to `UpdateCommand` with `if_not_exists` on enrichment fields (`imageUrl`, `genres`, `bio`, `color`), so scraped metadata is never overwritten

### Frontend (`src/components/ArtistCard.jsx`, `src/pages/ArtistDetail.jsx`)
- Already reads `artist.imageUrl` first before falling back to Wikipedia API
- No frontend changes needed — photos appear automatically once DynamoDB is populated

---

## Result

- 999/999 artists enriched with Deezer photos on first run
- Image enrichment cost on subsequent runs: ~2s (DynamoDB scan only, no Deezer calls)
