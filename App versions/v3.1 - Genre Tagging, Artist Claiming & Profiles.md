# GigRadar — Version 3.1: Genre Tagging, Artist Claiming & Profiles

**Date:** 2026-04-04
**Deployment:** Pushed to `main` (commit `86fc9d3`). API Lambda updated directly via CLI.

---

## Overview

GigRadar v3.1 delivers the deferred items from v3.0: an internal admin dashboard for genre tagging and claim review, an artist claiming flow that lets band members take ownership of their profile, JWT-authenticated profile editing for claimed artists, and a verified badge shown on artist cards and detail pages.

---

## What Was Built

### `lambda/api/index.js`
- Added `UpdateCommand` to DynamoDB imports
- Updated CORS to allow `POST`, `PATCH`, `Authorization`, and `x-admin-key` headers
- Added `isAdmin(event)` — checks `x-admin-key` header against `ADMIN_API_KEY` env var
- Added `getJwtSub(event)` — base64-decodes Cognito ID token payload to extract `sub`
- **New routes:**
  - `GET /admin/artists` — all artists sorted by rank (admin key required)
  - `POST /admin/artists/:id/genres` — set genres array on artist (admin key required)
  - `GET /admin/claims` — artists with `pendingClaim` attribute (admin key required)
  - `POST /admin/claims/:id/approve` — set `claimedBy`, `verified: true`, remove `pendingClaim` (admin key required)
  - `POST /admin/claims/:id/reject` — remove `pendingClaim` (admin key required)
  - `POST /artists/:id/claim` — submit claim with email + optional note (JWT auth, 409 if already claimed)
  - `PATCH /artists/:id` — update bio/imageUrl/website/spotify/instagram/facebook (JWT auth, must be `claimedBy` sub)
- `ADMIN_API_KEY` env var set on Lambda

### `src/utils/cognito.js`
- Added `getToken()` — returns the Cognito ID token JWT string for the current session

### `src/utils/api.js`
- Added `post()` and `patch()` helpers (include `Authorization: Bearer <token>` header)
- Added `adminGet()` and `adminPost()` helpers (include `x-admin-key` header)
- Added: `claimArtist`, `updateArtist`, `adminGetArtists`, `adminSetGenres`, `adminGetClaims`, `adminApproveClaim`, `adminRejectClaim`

### `src/components/ArtistCard.jsx`
- Verified badge (brand-coloured checkmark SVG) shown next to artist name when `artist.verified === true`

### `src/pages/ArtistDetail.jsx`
- **Verified badge** — pill badge next to artist name ("Verified artist") when `artist.verified === true`
- **Social links** — website, Spotify, Instagram, Facebook links shown when present
- **Edit mode** — pencil button shown to the artist owner (`user.sub === artist.claimedBy`); inline form for bio, imageUrl, website, Spotify, Instagram, Facebook; saves via `PATCH /artists/:id` with JWT
- **Claim flow** — "Is this your band? →" link shown for unclaimed artists (prompts sign-up if logged out); opens `ClaimModal`; confirmation message shown after submission

### `src/components/ClaimModal.jsx` (new)
- Prefills user email (read-only), optional note textarea
- Submits via `POST /artists/:id/claim` with JWT token
- Handles "already claimed" 409 error

### `src/pages/Admin.jsx` (new) — `/admin`
- Key entry screen (stored in `sessionStorage`); auto-validates on load if key exists
- **Genre Queue tab**: lists all artists ordered by Last.fm rank; filter toggle for untagged only; click any artist to open genre edit panel; 15-genre grid (same taxonomy as Discover); saves via `POST /admin/artists/:id/genres`; updates list in-place without reload
- **Claims tab**: lists pending claims with email, date, and optional note; approve/reject buttons; approved claims set `verified: true` on the artist record

### `src/App.jsx`
- Added route: `/admin`
