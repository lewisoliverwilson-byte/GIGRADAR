# GigRadar — Version 2.0: React Rewrite & Multi-Source Scraper

**Date:** 2026-04-04
**Deployment:** Pushed to `main` — Amplify redeploy (commit `eec0339`). Both Lambda functions updated directly via CLI.

---

## Overview

Complete rebuild per the GigRadar Technical Specification. Replaced the vanilla JS frontend with a React 18 + Vite + Tailwind CSS app. Expanded the gig scraper from 1 source (Ticketmaster) to 10 sources, with cross-source deduplication by artist + date + venue.

---

## Frontend Changes

### Stack
- **React 18** + **Vite** (replaces static `public/` served by Express)
- **Tailwind CSS** (dark mode only, custom design tokens)
- **React Router v6** (client-side routing)
- Amplify build now runs `npm ci && npm run build` → `dist/`

### Pages
| Route | Component | Description |
|---|---|---|
| `/` | `Home` | Hero, followed artists' gigs, top artists grid, upcoming gigs |
| `/artists` | `Artists` | Searchable, filterable grid of all 1,000 artists |
| `/artists/:id` | `ArtistDetail` | Artist hero, upcoming/past gig tabs |
| `/gigs` | `Gigs` | All upcoming gigs, filterable by city and following |
| `/profile` | `Profile` | Followed artists list, account info |

### Components
- `ArtistCard` — artist photo (Wikipedia API), initials fallback, follow button, upcoming count
- `GigCard` — date strip, venue, city, price, sold-out badge, source attribution badges, ticket link
- `FollowButton` — optimistic toggle; shows account prompt for guests
- `AuthModal` — login/signup tabs + email verification step
- `AccountPrompt` — modal gate for follow/calendar/notifications
- `Navbar` — sticky, with user menu and search shortcut

### State
- `AuthContext` — current Cognito user, login/logout, auth modal open state
- `FollowContext` — followed artist IDs in `localStorage`, per-user keyed by Cognito sub

## Scraper Changes (`lambda/scraper/index.js`)

### Sources (10 total)
| # | Source | Method | Notes |
|---|---|---|---|
| 1 | Last.fm | API | Ranking signal — top 1000 UK artists |
| 2 | Ticketmaster | API | Have API key; major venues |
| 3 | Bandsintown | Free API | No key needed (`app_id=gigradar`); indie/mid-size |
| 4 | Skiddle | Free API | Requires `SKIDDLE_API_KEY` env var; UK clubs & gigs |
| 5 | Songkick | Scrape (JSON-LD) | Broad listings; polite 400ms delay |
| 6 | Dice.fm | Scrape (REST) | London indie/electronic |
| 7 | Resident Advisor | GraphQL | Electronic music |
| 8 | See Tickets | Scrape | UK-wide; HTML extraction |
| 9 | Gigantic | Scrape | UK indie/alt |
| 10 | WeGotTickets | Scrape | Grassroots venues |
| 11 | Eventbrite | Scrape (JSON blob) | Smaller events |

### Deduplication
Gigs from all sources are merged on a `dedupKey = artistId|date|normalisedVenueName`. When a duplicate is found:
- Ticket links from all sources are aggregated (no duplicates by URL)
- Source attribution list is merged
- Support acts are merged (unique set)
- `isSoldOut` is `false` if any source reports availability
- `doorsTime` uses first non-null value

## Known Issues / Limitations

1. **Skiddle key not yet set** — add `SKIDDLE_API_KEY` to `gigradar-scraper` env vars for extra coverage.
2. **Scraped sources are best-effort** — if Songkick/Gigantic/See Tickets change their HTML structure, those fetchers return 0 results silently.
3. **Bandsintown sampled at 300 artists** — full 1000 would take ~4 minutes at 200ms/request; current cap keeps scraper within Lambda timeout.
4. **No Spotify/Apple Music import yet** — onboarding flow planned for v2.1.
5. **No Google/Apple SSO yet** — planned for v2.1.
6. **`/settings/notifications` page not built** — linked from profile but returns 404 for now.
