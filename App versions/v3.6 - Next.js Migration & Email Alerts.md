# v3.6 — Next.js Migration, Email Alerts & Gig Filters

**Released:** 2026-04-18  
**Commit:** 6c167b7

---

## What Changed

### Next.js 14 Migration (Vite → Next.js Pages Router)
- Replaced Vite + React Router SPA with Next.js 14 (Pages Router)
- `src/views/` holds all component logic; `src/pages/` holds thin Next.js route wrappers
- SSR via `getServerSideProps` on `/artists/[id]` and `/venues/[slug]` for proper meta tags and SEO
- AWS Amplify platform updated to `WEB_COMPUTE` (required for SSR/Next.js)
- `amplify.yml` updated: `baseDirectory: .next`, cache paths updated
- Fixed Windows case-insensitive filesystem conflict by separating views from pages
- `admin` page rendered client-side only (uses `sessionStorage`)

### Email Gig Alerts
- New `lambda/notify/index.js` — triggered by DynamoDB Streams on `gigradar-gigs` table
- Queries `gigradar-follows` for artist/venue followers and sends SES email alerts
- New `AlertButton` component on artist and venue pages — prompts for email, stores in `localStorage`
- New API endpoints: `POST /follows`, `DELETE /follows`, `GET /follows/check`, `GET /unsubscribe`
- Unsubscribe link in every alert email uses `unsubToken` stored in `gigradar-follows`

### City & Date Filters on /gigs
- City dropdown (20 UK cities), date range pickers (from/to)
- Filters sync to URL query params via `router.replace` (shallow routing)
- Lambda `getGigs()` supports `city`, `from`, `to` params with full DynamoDB pagination

### Infrastructure
- DynamoDB Streams (NEW_IMAGE) enabled on `gigradar-gigs`
- New DynamoDB table: `gigradar-follows` (GSIs on `email` and `targetId`)
- IAM policy updated: DynamoDB Streams perms, SES perms, `gigradar-follows` access
- SES domain verification pending for `gigradar.co.uk` (TXT record needed)
