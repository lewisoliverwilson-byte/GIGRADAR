## Critical Bug Fix

### Spotify Matching Fix (matchArtists — was returning wrong results)
- **Root cause**: `matchArtists` in the API used a single non-paginated `ScanCommand`. DynamoDB returns max 1MB per scan. At 47k artists, only ~13k were visible per call — 75% of artists were invisible.
- **Fix**: Replaced with paginated scan loop using `LastEvaluatedKey` (same pattern as all other scan functions).
- **Side effect fix**: Opportunistic save now stores `spotifyId` field (not just URL), image, and genres when a Spotify match is made during connection.
- **New artist stubs** now include `spotifyId` field.

## Task 1: Spotify Enrichment at Scale

### `scripts/enrich-artists-spotify-v2.cjs` (new)
Full bulk enrichment for all 47k artists against Spotify.
- 10 concurrent workers (~300 req/sec)
- Strict exact name matching (normalised, no partial matches)
- Skips zero-follower results (almost always wrong matches)
- Stores `spotifyId` separately (O(1) lookup vs extracting from URL)
- Fetches highest-res Spotify image (640px) — stored as `imageUrl` if not already set
- Always updates `spotifyPopularity` and `monthlyListeners`
- `if_not_exists` for genres (Last.fm genres take priority)
- Deduplication: in-memory Set prevents the same Spotify ID being assigned to two artists
- Sorts upcoming-gigs artists first (most impactful first)
- Resume support via `spotify-v2-progress.json`
- Commands: `node scripts/enrich-artists-spotify-v2.cjs` or `--quick` (upcoming only) or `--resume`

**Note**: Spotify API rate-limited after parallel testing runs (~20h ban on client ID). Re-run after 2026-04-26 14:00 UTC.

## Task 6: Price Data

### `lambda/scraper/index.js` — minPrice field
Added numeric `minPrice` field to all scrapers:
- **Ticketmaster** (per-artist, bulk, venue): `minPrice: ev.priceRanges?.[0]?.min`
- **Skiddle** (search, venue): `minPrice: parseFloat(ev.mineticketprice)`
- **Venue stored URLs** (JSON-LD): `minPrice: parseFloat(ev.offers[0].price)`

### `scripts/backfill-min-price.cjs` (new)
Backfilled 19,495 existing gigs with numeric `minPrice` parsed from price strings (e.g., "£15" → 15). Takes price coverage from ~4% to ~26% of the 87k gig database.

## Task 7: onSaleDate Fix

### `lambda/scraper/index.js` — onSaleDate field
Added `onSaleDate` to all scrapers:
- **Ticketmaster**: `ev.sales?.public?.startDateTime?.split('T')[0]`
- **Skiddle**: `ev.onsaledate?.split('T')[0]`
- **Venue stored URLs**: `ev.offers[0].validFrom?.split('T')[0]`
