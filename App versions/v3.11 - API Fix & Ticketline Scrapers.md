## Critical API Fixes

### 1. Full table scan with 2,000-item cap → GSI-based date queries
**File:** `lambda/api/index.js` (`getGigs` function)

The old implementation scanned the entire gigs table but stopped at 2,000 items for
safety. With 119k+ upcoming gigs, users were seeing a random 2,000-item slice. Fixed
by querying the `date-index` GSI (HASH=date) for each date in the requested range,
in parallel batches of 5. Now returns all gigs for the requested date window (up to
90 days), correctly sorted by date.

### 2. City filter broken — wrong field name + case mismatch
**File:** `lambda/api/index.js`

The city filter used `contains(city, ...)` but the DynamoDB attribute is `venueCity`.
Separately, query params were lowercased ("manchester") but cities are stored as
title-case ("Manchester"). Both bugs fixed: city param is now title-cased before
filtering, and the filter uses `venueCity`.
