# GigRadar — Version 2.4: Worldwide Expansion

**Date:** 2026-04-04
**Deployment:** Pushed to `main` — Lambda updated directly via CLI (commit `3ea50f4`).

---

## Overview

GigRadar now covers artists and gigs worldwide, not just the UK. The artist list switches from the UK Last.fm chart to the global top 1,000. All major scraper sources have country filters removed. UK-only ticketing platforms (Skiddle, See Tickets, Gigantic, WeGotTickets) remain unchanged — they still contribute UK data for any UK gigs in the global list.

---

## What Changed

### `lambda/scraper/index.js`

**Last.fm**
- Changed from `geo.getTopArtists&country=united+kingdom` → `chart.getTopArtists`
- Now fetches the global top 1,000 most-listened-to artists (not UK-specific chart)
- Response key changed from `topartists.artist` to `artists.artist` (with fallback)

**Ticketmaster**
- Removed `countryCode=GB` — now scans worldwide music events
- Price formatting updated: uses `currSym()` helper for correct currency symbol

**Bandsintown**
- Removed `['GB', 'UK', 'United Kingdom']` country filter
- `venueCountry` now stores actual country from API response

**Songkick**
- Removed `['GB', 'UK']` country filter
- `venueCountry` now stores `location.addressCountry` from JSON-LD

**Dice**
- Removed `country_codes[]=GB` from URL — now fetches worldwide Dice events
- `venueCountry` uses `venue.country_code || venue.country`
- Price uses actual currency from `ev.currency`

**Resident Advisor**
- Removed `areas: { id: 'GB' }` from GraphQL query variables — worldwide events
- Removed secondary `country !== 'GB'` filter
- `venueCountry` uses `ev.venue.area.country.isoCode`
- Price uses currency from `priceRange.currency`

**Eventbrite**
- URL changed from `eventbrite.co.uk/d/united-kingdom/music/` to `eventbrite.com/d/music/`
- `venueCountry` uses `ev.venue.address.country`
- Price uses currency from `minimum_ticket_price.currency`

**Setlist.fm**
- Removed `country !== 'GB'` filter — now stores setlists from all countries
- `venueCountry` uses `city.country.code` from Setlist.fm response

**Added `currSym()` helper**
- Maps ISO currency codes (USD, GBP, EUR, AUD, CAD, etc.) to display symbols
- Used by Ticketmaster, Dice, RA, and Eventbrite

**`upsertArtist`**
- Default `country` field changed from `'UK'` to `''` (artists are now global)

### `src/pages/Home.jsx`

- Hero headline: "Every UK gig." → "Every gig."
- Subtitle: "for the top 1,000 UK artists" → "for the top 1,000 artists worldwide"
- Stats bar: "UK artists" → "artists worldwide"
- Section header: "Top UK artists with upcoming gigs" → "Top artists with upcoming gigs"

## Known Limitations

1. **Artist list transition**: Artists in DynamoDB from the old UK chart will remain until the global chart replaces them. The global top 1,000 overlaps heavily with the old UK list but adds more US/international acts.
2. **Skiddle / See Tickets / Gigantic / WeGotTickets**: These are UK-only platforms and will only contribute UK gigs. No change needed — they add value for any UK acts in the global list.
3. **Data volume**: Worldwide coverage will significantly increase the number of gigs in DynamoDB over the next few scraper runs. The `getGigs()` API scan limit (500) ensures response times stay reasonable.
4. **Setlist.fm rate limit**: Removing the UK filter means all setlists worldwide are now stored. The 1,440 req/day limit is unchanged — same number of API calls, just more data per response.
