# v3.3 — Spotify Integration

**Released:** 2026-04-05  
**Commits:** ff0c7f7, 8ead941, 0223c8f

## What's New

### Spotify OAuth onboarding flow
- New `/onboarding/connect` screen shown to all new signups after email verification
- PKCE-based OAuth flow — token exchange happens directly in the browser (no client secret exposed)
- Redirect URI handled at `/auth/spotify/callback`

### Artist import & confirmation
- Fetches up to 100 top artists from Spotify (`long_term` listening history, 2 pages)
- Matches against GigRadar UK artist database by Spotify ID or normalised name
- `/onboarding/artists` confirmation screen — pre-checked artist list, select/deselect controls, "Follow X artists" CTA
- Empty state if no UK artists found

### Profile — Connected Accounts section
- `/profile` now shows Spotify connection status with connected date
- Connect button for unconnected users
- Disconnect with confirmation dialog (follows preserved on disconnect)
- Apple Music placeholder (coming soon)

### Backend
- New Lambda endpoint: `POST /api/artists/match` — takes list of Spotify artist names/IDs, returns matched GigRadar artists
- `gigradar-spotify-tokens` DynamoDB table for token storage
- Amplify SPA rewrite rule configured (`</[^.]+/>` → `/index.html`)

## Infrastructure Changes
- Amplify custom rewrite rule set via AWS CLI
- API Gateway: added POST, OPTIONS, PATCH routes
- API Gateway CORS updated to allow `Authorization` header and POST/PATCH methods
- Lambda env vars: `SPOTIFY_CLIENT_ID`, `SPOTIFY_CLIENT_SECRET`
