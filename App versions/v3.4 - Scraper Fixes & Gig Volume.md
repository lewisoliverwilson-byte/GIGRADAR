# v3.4 — Scraper Fixes & Gig Volume

**Date:** 2026-04-18

## Summary

Fixed two broken scraper sources that had silently stopped working, resulting in a 2.4× increase in gigs ingested per scraper run (609 → 1,449 unique gigs per run).

## What Changed

### Skiddle — Pagination Fix
- **Problem:** Skiddle's API uses `offset=` for pagination, but the scraper was using `page=`. The `page` parameter was silently ignored, returning page 1 on every request. The duplicate-ID detection stopped the scraper at page 2 every run.
- **Fix:** Switched to `offset=N` (offset=0, 100, 200...) with a cap of 1,000 events per run.
- **Result:** 89 → **917 gigs per run** (~10× improvement).

### Dice.fm — API Replaced
- **Problem:** `api.dice.fm` REST endpoint returning 404 (API decommissioned).
- **Fix:** Switched to scraping Next.js SSR JSON (`__NEXT_DATA__`) from `dice.fm/browse` pages. Extracts `summary_lineup.top_artists` and `date_unix` from the embedded page props.
- **Result:** 0 → **22 gigs per run**.

### Resident Advisor — Diagnosed
- **Finding:** RA's GraphQL API works from local IPs but returns 0 listings from Lambda (us-east-1). Cloudflare/WAF blocks AWS datacenter IPs. Added HTTP status logging to confirm. Not fixed — would require a proxy to solve.
