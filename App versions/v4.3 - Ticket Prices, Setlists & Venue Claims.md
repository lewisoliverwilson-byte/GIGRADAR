# v4.3 ‚Äî Ticket Prices, Setlists, On-Sale Tracking & Venue Claims

**Date:** 2026-04-21

## Features Shipped

### Ticket Prices ‚Äî `minPrice` Field Across All Scrapers
- `minPrice` (number, GBP) added to all 14 scrapers: TM, Skiddle, DICE, Ticketline, Fatsoma, SeeTickets, Eventbrite, WeGotTickets, RA, Songkick, Gigantic, Bandsintown, Ents24, venue-gigs
- Sources: raw API data (pence‚Üípounds for DICE/Fatsoma), schema.org offers, HTML regex (Ents24)
- Free events set `minPrice: 0` (Eventbrite, Bandsintown)
- GigCard now shows "from ¬£X" badge when `minPrice` is known and no per-ticket price is shown
- GigCard shows "Free" badge in emerald for `minPrice === 0`

### Price Filter in Gigs Page
- New dropdown: Any price / Free / Under ¬£10 / Under ¬£20 / Under ¬£30 / Under ¬£50
- Client-side filter on `minPrice` field; only shows gigs with known prices at or below cap
- Filter shown in count display: "124 gigs ¬∑ under ¬£20"

### On-Sale Tracking
- `onSaleDate` field added to Ticketmaster scraper (`ev.sales.public.startDateTime`)
- Lambda: `GET /on-sale` ‚Äî gigs where `onSaleDate` falls in the next 7 days
- Lambda: `GET /coming-soon` ‚Äî gigs with `lastUpdated` in the last 14 days
- Homepage: "Tickets on sale this week" section (amber, with pulsing dot)
- Homepage: "Just announced" section (sky blue) ‚Äî recently added upcoming gigs
- Both sections hidden when empty

### Setlists Tab ‚Äî Setlist.fm Integration
- `GET /artists/:id/setlists` ‚Äî Lambda proxy to Setlist.fm API using `mbid` (already stored)
- `SETLISTFM_KEY` env variable on Lambda required; gracefully returns `[]` if missing
- ArtistDetail.jsx: new "Setlists" tab alongside Upcoming/Past
- Lazy-loaded on first click (no fetch on page load)
- Shows: venue, city, country, tour name, date, full set list with song numbers
- Cover songs and tape-only songs indicated

### Venue Claim Flow
- `POST /venues/:slug/claim` ‚Äî submit a claim (requires JWT), stores `pendingClaim` object
- `PATCH /venues/:slug` ‚Äî update venue profile once claimed (bio, website, Instagram, Facebook, bookingEmail, imageUrl, capacity, isGrassroots)
- `GET /admin/venue-claims` + approve/reject endpoints
- VenuePage: "Is this your venue?" button ‚Üí opens VenueClaimModal
- VenueClaimModal: email, role dropdown (Owner / Promoter / Booking agent / Staff), optional note
- VenuePage: edit form for claimed owners (same fields as patch endpoint)
- VenuePage: Grassroots badge + Verified badge on venue header
- Admin.jsx: new "Venue Claims" tab with approve/reject buttons

### API
- `GET /gigs?maxPrice=` ‚Äî filter by `minPrice <= maxPrice` (post-scan JS filter)
- `GET /on-sale?city=&genre=`
- `GET /coming-soon?city=&genre=`
- `GET /artists/:id/setlists`
- `POST /venues/:slug/claim`
- `PATCH /venues/:slug`
- `GET /admin/venue-claims`
- `POST /admin/venue-claims/:id/approve`
- `POST /admin/venue-claims/:id/reject`
