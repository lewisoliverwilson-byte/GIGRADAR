# GigRadar — Version 1.1: Artist Images

**Date:** 2026-04-03
**Deployment:** Pushed to `main` — deployed via AWS Amplify (commit `a26192f`)

---

## Overview

Added real artist photos across the site, fetched from the Wikipedia REST API. Artist cards and the artist detail hero now show actual photos in place of the colour gradient placeholders, with the gradient retained as a fallback.

---

## What Changed

### Data (`data.js`)
- Added `wikipedia` property to all 20 artists — the Wikipedia article title used to fetch the thumbnail (e.g. `'Arctic_Monkeys'`, `'Fontaines_D.C.'`, `'Bicep_(band)'`)

### App (`app.js`)
- **`artistCardHTML`** — avatar div now carries a `data-wikipedia` attribute when available
- **`renderArtist`** — hero avatar (`.artist-avatar-lg`) and blurred background (`.artist-hero-bg`) both carry `data-wikipedia` attributes; initial letter wrapped in `<span class="artist-initial">` for consistent targeting
- **`loadArtistImages()`** — new async function that finds all `[data-wikipedia]` elements, fetches thumbnails from `https://en.wikipedia.org/api/rest_v1/page/summary/{title}`, and injects a `<img class="artist-img">` (or sets `backgroundImage` on the hero bg). Results are cached in `imageCache` to avoid re-fetching on navigation.
- **`router()`** — calls `loadArtistImages()` after every view render

### Styles (`main.css`)
- **`.artist-img`** — `position: absolute; inset: 0; width/height: 100%; object-fit: cover; object-position: center top`
- **`.artist-avatar-lg`** — added `position: relative; overflow: hidden` so the injected image is correctly clipped to the avatar shape
- **`.artist-initial`** — added `transition: opacity 0.3s`; hidden via JS once an image loads successfully

---

## Behaviour

- Images load asynchronously after the page renders — the gradient shows briefly then fades to the photo
- If Wikipedia returns no thumbnail (or the fetch fails), the gradient and initial letter remain unchanged
- Images are cached in memory for the session — navigating back to a page does not re-fetch

---

## Known Issues / Limitations

Carries forward all issues from v1.0, plus:

1. **Wikipedia thumbnail quality varies** — some artists have high-quality press photos; others have lower-resolution or awkwardly cropped images depending on what Wikipedia's article uses.

2. **No persistent image cache** — the in-memory cache is cleared on page refresh, causing a fresh round of API calls each session.

3. **Wikipedia article titles are hardcoded** — if a Wikipedia article is renamed or disambiguation changes (e.g. `Bicep_(band)` → `Bicep_(duo)`), the image will silently stop loading and fall back to the gradient.
